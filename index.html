¿Qué es un Proyecto? (En Profundidad)

Es un esfuerzo temporal con un principio y un fin, que crea un resultado único.

La principal diferencia con las Operaciones es que estas son continuas y repetitivas (ej: dar soporte técnico), mientras que un proyecto es un evento único (ej: crear un nuevo software de soporte).

Tipos de Proyectos en la Industria Tech

  • Desarrollo de Software: Crear una aplicación móvil, una plataforma web o un sistema de escritorio como el nuestro.
  • Infraestructura: Migrar servidores a la nube (AWS, Azure), configurar redes o mejorar la ciberseguridad.
  • Investigación y Desarrollo (I+D): Explorar la viabilidad de usar una nueva tecnología (ej. Blockchain, IA) para un producto futuro.
  • Análisis de Datos: Implementar un dashboard de Business Intelligence o crear un modelo predictivo.

Las 6 Etapas del Ciclo de Vida de un Proyecto

Todo proyecto, sin importar su tamaño o tipo, atraviesa un ciclo de vida estandarizado.

Iniciación → Planificación → Ejecución → Monitoreo y Control → Cierre

Conocer estas etapas nos permite saber qué hacer y qué esperar en cada momento.

Etapa 1: Iniciación

Objetivo: Definir el proyecto a un alto nivel y obtener la autorización para comenzarlo.

Aquí se responde: ¿Es este proyecto viable? ¿Qué problema resuelve?

  • Actividades Clave: Identificar stakeholders, analizar viabilidad, definir el objetivo general.
  • Entregable Principal: Acta de Constitución del Proyecto (Project Charter).

Etapa 2: Planificación

Objetivo: Crear una hoja de ruta detallada que guiará a todo el equipo.

Esta es la fase más crítica. Un mal plan casi siempre lleva a un mal resultado.

  • Actividades Clave: Crear el WBS (Work Breakdown Structure), estimar tiempos y costos, planificar recursos, identificar riesgos.
  • Entregable Principal: Plan de Gestión del Proyecto.

Etapa 3 y 4: Ejecución y Monitoreo

Estas dos etapas ocurren al mismo tiempo.

Ejecución: Es donde "se hace el trabajo". El equipo de desarrollo escribe código, los diseñadores crean interfaces, etc.

Monitoreo y Control: Es donde se mide el progreso contra el plan. ¿Vamos a tiempo? ¿Nos estamos pasando del presupuesto? ¿Han surgido nuevos riesgos?

Actividades: Reuniones de seguimiento, reportes de estado, gestión de cambios.

Etapa 5: Cierre

Objetivo: Finalizar formalmente todas las actividades del proyecto.

No se trata solo de entregar el producto, sino de cerrar el proceso de forma ordenada.

  • Actividades Clave: Entregar el producto final, obtener la aceptación del cliente, liberar al equipo del proyecto, documentar las lecciones aprendidas.
  • Entregable Principal: Producto Final y Reporte de Cierre del Proyecto.

Metodologías: Tres Enfoques

Waterfall (Cascada): Un modelo lineal y secuencial. Cada fase debe completarse antes de iniciar la siguiente. Es rígido y poco adaptable a cambios.

Definición → Planificación → Desarrollo → Pruebas → Despliegue

Agile (Ágil): Un enfoque iterativo e incremental. El trabajo se divide en ciclos cortos (sprints) que producen una versión funcional del producto. Abraza el cambio.

Idea → Sprint 1 → Producto v1 → Sprint 2 → Producto v2 ...

Fase 1: La Definición

Aplicando la teoría en nuestro proyecto

Definiendo Objetivos SMART

Un objetivo bien definido evita ambigüedades. El framework SMART es el estándar de la industria.

  • Specific (Específico): Claro y sin dudas.
  • Measurable (Medible): Cuantificable.
  • Achievable (Alcanzable): Realista con los recursos disponibles.
  • Relevant (Relevante): Importante para el negocio o el cliente.
  • Time-bound (Limitado en tiempo): Con una fecha límite clara.

Ejercicio Práctico 1: Definición

Objetivo General: "Crear un sistema de gestión de empleados en Python".

Tomando el objetivo general de nuestro proyecto, reescríbelo como un objetivo SMART para la primera versión (MVP) que se completará al final del bootcamp.

Solución Ejercicio 1 (Ejemplo)

Un posible objetivo SMART podría ser:

(S) Específico: Desarrollar un programa de consola en Python que gestione empleados, jefes y áreas, permitiendo agregar, consultar y guardar datos en archivos de texto.

(M) Medible: El sistema debe cumplir con todas las funcionalidades descritas en el PDF del proyecto.

(A) Alcanzable: Utilizando los conocimientos de Python y POO adquiridos en el bootcamp.

(R) Relevante: Es el proyecto final requerido para aplicar y demostrar las competencias del curso.

(T) Limitado en tiempo: El proyecto debe ser entregado en la fecha de finalización del bootcamp.

Fase 2: La Planificación

"¿Cómo lo vamos a construir?"

La Herramienta Clave: WBS

El Work Breakdown Structure (Desglose de Tareas) consiste en dividir un gran proyecto en partes más pequeñas y manejables.

Esto reduce la complejidad y facilita la asignación y estimación de tareas.

Toma el requisito "Gestionar Áreas" del proyecto y crea un WBS detallado. Incluye al menos 4 sub-tareas técnicas necesarias para implementarlo.

Solución Ejercicio 2 (Ejemplo)

   ↳ Tarea 3: Persistencia de datos
      ↳ Sub-tarea 3.1: Modificar función de guardado para incluir áreas.

Fase 3: Estimación y Riesgos

"¿Cuánto tardará y qué podría salir mal?"

Una introducción a la Gestión de Riesgos

Un riesgo es un evento incierto que, si ocurre, tendrá un impacto (positivo o negativo) en el proyecto. Identificarlos temprano es clave.

"No podemos predecir el futuro, pero podemos prepararnos para él."

  • Riesgos Técnicos: ¿La librería que queremos usar es estable? ¿Entendemos bien la persistencia de datos?
  • Riesgos de Equipo: ¿Alguien del equipo tiene menos experiencia? ¿Hay riesgo de falta de comunicación?
  • Riesgos Externos: ¿Los requisitos podrían cambiar a mitad del desarrollo?

Ejercicio Práctico 4: Estimación y Riesgos

Tarea: "Implementar la persistencia de datos (guardar y cargar desde archivos de texto)".

1. Asigna una estimación de tiempo (en horas) a esta tarea.

2. Identifica al menos un riesgo técnico asociado a esta tarea.

Solución Ejercicio 3 (Ejemplo)

1. Estimación de tiempo:

2.5 horas. Incluye investigar el formato (CSV, JSON), escribir el código para guardar, escribir el código para cargar y probar que los datos no se corrompan.

2. Riesgo Técnico Identificado:

Riesgo: El formato de los datos guardados podría corromperse o ser inconsistente, lo que impediría cargar la información correctamente al reiniciar el programa.
Mitigación: Implementar un manejo de errores robusto (`try-except`) durante la lectura del archivo y definir un formato de datos estricto desde el principio.

Nota sobre Indicadores Financieros (VAN/TIR)

En proyectos corporativos, las estimaciones de tiempo se convierten en costos (salarios de desarrolladores).

Estos costos se comparan con los beneficios esperados del proyecto usando indicadores como:

  • VAN (Valor Actual Neto): Mide la rentabilidad del proyecto en dinero de hoy. Si VAN > 0, es viable.
  • TIR (Tasa Interna de Retorno): El % de ganancia que genera el proyecto. Se compara con una tasa de interés de referencia.

Aunque no los calcularemos, es fundamental conocer su existencia en el mundo profesional.
